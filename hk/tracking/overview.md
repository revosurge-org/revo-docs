# 追蹤概述

**對象：** 開發者、追蹤/BI 工程師、UA 經理（技術向）

追蹤將廣告投放與可衡量的成效連結。在 RevoSurge 中，有兩種主要方式發送事件：
- **Web 追蹤器（瀏覽器端）**
- **伺服器到伺服器 (S2S) 事件（後端）**

## 本文內容
- 追蹤的用途
- Web 追蹤器 vs S2S（何時使用哪種）
- 事件基礎
- 追蹤在產品中的體現

## 追蹤的用途
追蹤支援：
- 轉化衡量（如註冊、充值）
- 歸因（將成效與廣告系列連結）
- 向目標事件優化（在啟用時）
- 基於行為及成效建立受眾細分

## Web 追蹤器 vs S2S（何時使用哪種）
**Web 追蹤器**適用於：
- 可在網站上部署程式碼
- 需要即時站內訊號（頁面瀏覽、表單提交等）

**S2S 事件**適用於：
- 轉化在後端發生（充值、訂單、帳戶狀態變更）
- 需要可靠的收入及交易數據
- 希望避免客戶端攔截

> 許多廣告主**同時使用**：Web 追蹤器用於站內行為，S2S 用於收入級事件。

## 事件基礎（通用）
**事件**是可衡量並可能優化的命名動作。  
常見示例：
- Register
- First Time Deposit (FTD)
- Deposit（重複）

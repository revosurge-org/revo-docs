# 廣告系列設定

**對象：** 在 AdWave 投放廣告的 UA 經理、廣告系列營運與增長團隊。

廣告系列設定是一個單頁、逐步進行的流程，帶你在同一處完成從建立產品到上線投放（含素材）的全過程。

## 前提條件 {#prerequisites}

廣告系列建立**受數據配置限制**。在 AdWave 引導頁（**接入你的數據**）上，*建立廣告系列* 會一直鎖定，直到你完成步驟 1–3。開始前請確保：

- **已建立產品。** 建立**產品**（你要推廣的應用或網站），並填寫其網域、追蹤器 ID 與充值匯率。
- **Web 追蹤器已啟用。** 在 Web 追蹤器收到首個事件之前，產品處於 **Inactive（未啟用）** 狀態；只有**已啟用**的產品才能在廣告系列流程中選擇。參見 [安裝 Web 追蹤器](/hk/tracking/web-tracker/install)。
- **轉化事件已追蹤。** 你用於優化的目標事件（如 Register、Deposit）必須已透過追蹤流入。參見 [追蹤 → 概述](/hk/tracking/overview)。
- **錢包已充值。** 你的**錢包**餘額充足。參見 [充值與錢包](/hk/growth/funding-wallet)。
- **素材已就緒。** 素材已按規格準備好。參見 [素材要求與範例](/hk/adwave/creative-requirements)。

> **提示：** 你可在**管理產品**中查看產品狀態——**Tracker Status（追蹤器狀態）** 欄顯示 Active / Inactive，*Data Setup（數據配置）* 顯示有多少事件處於啟用狀態。

## 1) 產品 {#_1-product}

選擇你要推廣的**產品**。每張卡片顯示產品名稱及其 URL。

> **注意：** 「未啟用的產品需先啟用才能投入使用。」 若所需產品尚未啟用，請先啟用，再返回本流程。

## 2) 目標與廣告格式 {#_2-objective-ad-format}

**廣告系列名稱**

輸入用於報表的可讀名稱。AdWave 建議一套命名格式以保持廣告系列一致：

> 建議格式：**地區 + 促銷 + 廣告格式 + 開始日期**——例如 `IN_Register_Display_20260126`。

**目標事件**

選擇本廣告系列要優化的轉化事件。該事件用於衡量轉化並計算廣告系列的 **CPA**。

- **Register**
- **Deposit**

**廣告格式**

選擇一種格式。每種格式的素材要求不同（見 [步驟 6](#_6-creative)），建議出價也不同：

- **Pop** — 以覆蓋層或新視窗形式出現的廣告。極為吸睛；適合促銷與以轉化為目標的廣告系列。
- **Push** — 直接推送到用戶裝置的通知式廣告。適合再互動與持續觸達用戶。
- **Display** — 放置在固定位置（如側邊欄或頁首）的橫幅式廣告。適合強曝光與廣泛的品牌認知。
- **Native** — 無縫融入內容資訊流的廣告。干擾更小、更具吸引力；適合追求更高點擊率。

## 3) 目標地區 {#_3-target-geo}

搜尋並選擇要定向的國家或地區。每個廣告系列最多可添加 **20 個**地區。

> **提示：** 保持地區與素材語言一致。若你定向多個需要不同語言的市場，請為每個市場準備對應的素材（或素材語言）。參見 [步驟 6：素材](#_6-creative)。

## 4) 出價與日預算 {#_4-bid-daily-budget}

**出價**

- **出價策略：** **CPM**（每千次展示成本）。
- **最高 CPM 出價：** 你願意為每千次展示支付的最高價格。

AdWave 會顯示隨廣告格式而變化的**建議最高出價**（例如撰寫本文時 Display ≈ \$0.04、Native ≈ \$0.03、Push ≈ \$0.11、Pop ≈ \$0.11）。

> **注意：** 「這是我們為本廣告系列建議的最高出價。競價中的實際出價可能低於此上限。」

**日預算**

以美元（USD）設定每日花費上限。

> **提示：** 將日預算最低設為**目標出價的 20 倍**，以確保投放穩定、結果有意義。這樣算法每天才有足夠數據來學習並有效優化效果。

## 5) 排期 {#_5-schedule}

- **開始日期：** 預設為現在；也可安排未來開始。
- **結束日期：** 選擇一個結束日期，或使用**快速選擇週期**從開始日期起設定 **7d / 14d / 30d / 60d / 90d** 的時長。

## 6) 素材 {#_6-creative}

> **注意：** 請先在 [步驟 2](#_2-objective-ad-format) 選擇廣告格式。素材欄位會隨格式而變化。

點擊 **添加素材（[格式]）** 以新增一個素材區塊。你可以**一次上傳多張圖片——最多 20 張**。

對每個素材：

- **素材語言：** 設定語言（如 EN）。勾選 **「本廣告系列的所有素材都使用此語言」** 可套用於每個素材。
- **目標 URL：** 選擇落地頁。勾選 **「本廣告系列所有素材共用一個 URL」** 可複用單個 URL，或使用 **添加落地頁** 註冊新的 URL。

各格式專屬欄位：

| 格式 | 素材 | 規格 |
| --- | --- | --- |
| **Pop** | 僅目標 URL | 無需圖片——Pop 直接呈現你的落地頁 |
| **Push** | 圖示 + 主圖 + 標題 + 描述 | 圖示：JPEG/PNG/JPG，192×192，≤ 200 KB。主圖：JPEG/PNG/JPG，≤ 720 KB，尺寸 492×328 / 360×240 / 720×480 |
| **Display** | 主圖 + 目標 URL | JPEG/PNG/JPG/GIF，≤ 500 KB，尺寸 300×250 / 320×50 / 728×90 / 468×60 / 160×600 / 300×100 |
| **Native** | 主圖 + 標題 + 描述 + 目標 URL | JPEG/PNG/JPG，≤ 500 KB，尺寸 300×250 / 300×100 / 320×50 / 728×90 / 468×60 / 160×600 |

> **提示：** 關於完整的素材指引、促銷／角度測試與範例，參見 [素材要求與範例](/hk/adwave/creative-requirements)。

## 發佈 {#launch}

沒有單獨的審核步驟。當每個部分都完成後，點擊（右上角的）**發佈** 即可啟用或排期廣告系列。

**發佈前檢查清單：**

- ✅ 已選擇產品（且已啟用）
- ✅ 廣告系列名稱遵循建議格式
- ✅ 已設定目標事件（Register / Deposit）
- ✅ 已選擇廣告格式
- ✅ 至少添加一個目標地區（≤ 20）
- ✅ 已設定最高 CPM 出價與日預算（預算 ≈ 出價的 20 倍）
- ✅ 已排期開始（及結束/週期）
- ✅ 素材已按規格添加，並設定語言與目標 URL

## 上線後：需監控的內容 {#after-launch-what-to-monitor}

**投放**

- 展示、花費、CPM
- 勝率（如可用）
- 地區分佈 vs 預期

**互動**

- 點擊、CTR
- 極高展示 + 幾乎零點擊 → 排查庫存質素

**成效**

- 轉化（Register / Deposit）
- CPA / ROAS（如已連接）

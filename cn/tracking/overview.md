# 追踪概述

**面向：** 开发者、追踪/BI 工程师、UA 经理（技术向）

追踪将广告投放与可衡量的成效关联。在 RevoSurge 中，有两种主要方式发送事件：
- **Web 追踪器（浏览器端）**
- **服务器到服务器 (S2S) 事件（后端）**

## 本文内容
- 追踪的用途
- Web 追踪器 vs S2S（何时使用哪种）
- 事件基础

## 追踪的用途
追踪支持：
- 转化衡量（如注册、充值）
- 归因（将成效与广告系列关联）
- 向目标事件优化（在启用时）
- 基于行为和成效构建受众细分

## Web 追踪器 vs S2S（何时使用哪种）
**Web 追踪器**适用于：
- 可在网站上部署代码
- 需要即时站内信号（页面浏览、表单提交等）

**S2S 事件**适用于：
- 转化在后端发生（充值、订单、账户状态变更）
- 需要可靠的收入和交易数据
- 希望避免客户端拦截

> 许多广告主**同时使用**：Web 追踪器用于站内行为，S2S 用于收入级事件。

## 事件基础（通用）
**事件**是可衡量并可能优化的命名动作。  
常见示例：
- Register
- First Time Deposit (FTD)
- Deposit（重复）
